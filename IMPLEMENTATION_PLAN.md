# Dmark 实施计划

## 目标

实现一个公网可访问的内部 Markdown API 文档浏览器：

- 管理员上传完整 Markdown API 文档
- 系统保存每次上传为一个历史版本
- 自动渲染 Markdown
- 自动生成可点击跳转的目录
- 每个项目生成不可猜的分享链接
- 前端通过分享链接查看最新文档，也可以切换历史版本

第一版不做自动 diff、不做在线编辑、不做用户体系、不做评论协作。

## 技术方案

默认技术栈：

- Next.js
- TypeScript
- Prisma
- SQLite
- react-markdown
- remark-gfm
- github-slugger
- rehype-sanitize

部署方式：

- 不使用 Docker
- 使用 Node.js 运行 Next.js 构建产物
- 使用 systemd 管理进程
- 使用 Caddy 或 Nginx 做 HTTPS 反向代理

如果后续改为 Vercel/serverless 部署，不要使用 SQLite，改用 Postgres。

## 核心功能

### 管理端

路由：

```text
/admin/login
/admin/projects
/admin/projects/new
/admin/projects/:id
/admin/projects/:id/upload
```

功能：

- 管理员登录
- 创建文档项目
- 查看项目列表
- 查看某个项目的版本列表
- 上传 `.md` 文件生成新版本
- 复制项目分享链接

管理员认证第一版使用环境变量：

```env
ADMIN_PASSWORD=change-me
SESSION_SECRET=change-me
DATABASE_URL=file:./data/dmark.db
```

不要第一版引入用户表、角色、注册登录。

### 分享端

路由：

```text
/docs/:token
/docs/:token?version=123
```

功能：

- 根据不可猜 token 查询项目
- 默认展示最新文档版本
- 支持历史版本下拉切换
- 渲染 Markdown 正文
- 从标题生成左侧目录
- 点击目录项跳转到对应位置
- 突出展示 `## 变更记录`

分享链接不需要登录。

## 数据模型

只建两张表。

```prisma
model Project {
  id        Int      @id @default(autoincrement())
  name      String
  slug      String   @unique
  shareToken String  @unique
  createdAt DateTime @default(now())
  updatedAt DateTime @updatedAt

  versions DocumentVersion[]
}

model DocumentVersion {
  id        Int      @id @default(autoincrement())
  projectId Int
  content   String
  note      String?
  createdAt DateTime @default(now())

  project Project @relation(fields: [projectId], references: [id], onDelete: Cascade)
}
```

说明：

- `shareToken` 使用随机 token，不要用项目名、ID 或时间戳生成。
- `slug` 仅用于管理端识别或内部展示，不用于公网分享。
- Markdown 原文直接存 `content`，第一版不拆结构化数据。

token 生成方式：

```ts
crypto.randomBytes(24).toString("base64url")
```

## Markdown 处理规则

渲染：

- 使用 `react-markdown`
- 启用 `remark-gfm` 支持表格
- 使用 `rehype-sanitize` 防止 Markdown 注入危险 HTML

目录：

- 从 Markdown 标题中提取目录
- 目录只展示 `##` 和 `###`
- 不展示 `#`，因为它是文档标题
- 不展示 `####`，避免请求参数、响应示例进入目录导致过碎

锚点规则：

- 去掉常见标点，例如 `.`
- 空格转为 `-`
- 与文档规范保持一致

示例：

```md
### 2. 更新当前用户资料
```

对应锚点：

```text
#2-更新当前用户资料
```

## 安全要求

必须做：

- 管理端登录保护
- 管理 cookie 设置 `httpOnly`
- 生产环境 cookie 设置 `secure`
- cookie 设置 `sameSite=lax`
- Markdown 渲染必须 sanitize
- 上传文件只接受 `.md`
- 限制上传大小，建议 2MB
- 分享 token 不可猜
- 不提供公开项目列表接口

第一版可以不做：

- 多用户
- 角色权限
- CSRF 完整方案
- 审批流

如果管理端后续暴露给多人使用，再补 CSRF 和用户体系。

## 实施步骤

### 1. 初始化项目

- 创建 Next.js + TypeScript 项目
- 配置 ESLint
- 添加 Prisma
- 添加 Markdown 渲染相关依赖

### 2. 建立数据库

- 创建 Prisma schema
- 建立 SQLite 数据文件目录
- 运行 migration
- 编写 Prisma client 封装

### 3. 实现认证

- `/admin/login`
- 校验 `ADMIN_PASSWORD`
- 写入签名 session cookie
- 管理端路由校验 cookie

不要创建用户表。

### 4. 实现项目管理

- 项目列表
- 创建项目
- 生成 `shareToken`
- 项目详情页展示版本列表和分享链接

### 5. 实现 Markdown 上传

- 上传 `.md`
- 校验文件扩展名和大小
- 读取 UTF-8 文本
- 保存为 `DocumentVersion`
- 上传成功后回到项目详情页

### 6. 实现文档分享页

- `/docs/:token`
- 查询项目
- 默认取最新版本
- 支持 `?version=123`
- 渲染 Markdown
- 生成目录
- 历史版本切换

### 7. 完善阅读体验

- 左侧目录跳转
- 当前阅读位置高亮可以后置；第一版不是必须
- 表格和代码块样式
- `变更记录` 区域高亮
- 移动端适配

### 8. 部署

推荐 VPS 部署：

- 安装 Node.js LTS
- 安装依赖
- 运行 Prisma migration
- `npm run build`
- 使用 systemd 启动 `npm start`
- 使用 Caddy 或 Nginx 配置 HTTPS 反向代理

生产环境变量：

```env
NODE_ENV=production
ADMIN_PASSWORD=strong-password
SESSION_SECRET=random-long-secret
DATABASE_URL=file:./data/dmark.db
```

SQLite 数据目录需要持久化并定期备份。

## 验收标准

功能验收：

- 管理员可以登录
- 可以创建项目
- 可以上传 Markdown
- 上传后生成新版本
- 分享链接不可猜
- 未登录用户可以通过分享链接访问文档
- 分享页默认展示最新版本
- 可以切换历史版本
- 左侧目录可以跳转到 `##`、`###` 标题
- `####` 不出现在目录里
- Markdown 表格、代码块、JSON 示例渲染正常

安全验收：

- 未登录不能访问 `/admin`
- 非 `.md` 文件不能上传
- 超过大小限制的文件不能上传
- Markdown 中的危险 HTML 不会执行
- 不存在公开项目列表接口

不在第一版验收范围：

- 自动 diff
- 在线编辑
- 评论
- 多用户权限
- OpenAPI 导入
- 接口调试
- 全文搜索

## 后续可选增强

只有第一版稳定使用后再考虑：

- 文档全文搜索
- 当前目录项滚动高亮

- 分享链接重置
